# 项目计划与里程碑

## 一、项目名称

AI赋能的非遗纹样智能生成与商用设计平台

## 二、项目目标

本项目面向非遗纹样的数字化保护与文创设计应用，尝试构建一套从数据集建设、生成模型微调到 Web 平台原型展示的完整流程。项目重点不是直接做成成熟商业产品，而是在大学生创新训练项目周期内，完成一个可运行、可展示、可验证的原型系统。

项目最终希望实现：

- 整理剪纸、敦煌壁画、苗绣、传统丝绸纹样等非遗纹样公开数据。
- 建立统一的数据清洗、标注和管理规范。
- 使用 SDXL + LoRA 微调生成特定非遗纹样风格。
- 补充 StyleGAN2 作为生成模型对比实验。
- 开发 Web 平台，实现纹样生成、参数调节、风格选择和图片导出。
- 通过校内用户测试验证平台原型的可用性。

## 三、研究内容

### 1. 非遗纹样数据集整合与优化

主要工作包括：

- 收集公开纹样图片。
- 按类别整理为剪纸、敦煌、苗绣、丝绸等目录。
- 删除重复、模糊、带明显水印或质量过低的图片。
- 统一图片尺寸、格式和命名规则。
- 为每张图片标注类别、风格、主题、颜色、来源和质量评分。

### 2. 生成模型选型与改进

主要工作包括：

- 学习 StyleGAN2、扩散模型、LoRA 微调的基本原理。
- 使用 SDXL + LoRA 跑通单风格微调流程。
- 尝试多风格 LoRA 或多个 LoRA 权重切换。
- 记录训练参数、生成效果和失败案例。
- 使用 StyleGAN2 进行对比实验，分析其在小样本纹样生成中的优缺点。

### 3. 商用设计平台原型开发

主要工作包括：

- 搭建前端页面，实现风格选择、提示词输入、参数调节和生成结果展示。
- 搭建后端接口，实现图片生成、历史记录和导出功能。
- 对接模型推理脚本。
- 设计适合文创场景的导出格式，例如 PNG、JPG。
- 补充文创预览功能，例如书签、明信片、帆布包等效果展示。

## 五、阶段计划

### 第 1 阶段：准备与调研（第 1-2 周）

任务：

- 明确项目目标、技术路线和成员分工。
- 学习 SDXL、LoRA、StyleGAN2 的基本概念。
- 搭建 Python、PyTorch、Node.js、Git 环境。
- 收集少量样例图片，完成初步分类。

阶段成果：

- 项目计划文档。
- 初版数据目录。
- 环境安装记录。
- 技术调研记录。

### 第 2 阶段：数据集建设（第 3-5 周）

任务：

- 每类收集 100-300 张公开图片。
- 完成去重、筛选、裁剪、尺寸统一。
- 建立标签 CSV/JSON。
- 划分训练集、验证集和测试集。

阶段成果：

- 可训练数据集。
- 数据集规范文档。
- 数据样例展示图。

### 第 3 阶段：模型实验（第 6-10 周）

任务：

- 跑通 SDXL + LoRA 单风格微调。
- 调整学习率、训练轮数、LoRA rank、分辨率等参数。
- 扩展到多个非遗风格。
- 跑通 StyleGAN2 对比实验。
- 建立人工评分表。

阶段成果：

- LoRA 权重文件。
- 生成样例图片。
- 模型对比结果。
- 实验记录文档。

### 第 4 阶段：平台原型开发（第 11-14 周）

任务：

- 完成前端工具页面。
- 完成后端 API。
- 接入模型推理脚本。
- 实现图片导出。
- 补充历史记录和结果预览。

阶段成果：

- 可运行 Web 原型。
- 平台截图。
- 演示视频或录屏。

### 第 5 阶段：测试与总结（第 15-16 周）

任务：

- 组织校内用户测试。
- 收集问卷和访谈反馈。
- 汇总用户评价。
- 完成最终报告和答辩 PPT。

阶段成果：

- 用户测试记录。
- 项目总结报告。
- 答辩材料。

## 六、最小可交付版本

如果时间紧张，优先完成以下内容：

- 数据集只做剪纸和敦煌两个类别。
- 模型只优先完成 SDXL + LoRA。
- 平台只实现类别选择、提示词输入、生成图片、PNG 导出。
- 用户测试先邀请 10 名校内同学体验。

完成最小版本后，再扩展苗绣、丝绸纹样、StyleGAN2 对比、文创预览等功能。

## 七、风险与应对

| 风险 | 可能表现 | 应对方案 |
| --- | --- | --- |
| 数据版权不清晰 | 图片不能公开展示 | 优先使用开放资源，记录来源和协议 |
| 数据质量不稳定 | 生成效果混乱 | 人工筛选高质量数据，先做小规模精品数据集 |
| 显存不足 | 模型训练失败 | 使用 LoRA、降低 batch size、降低分辨率 |
| 模型过拟合 | 生成结果重复 | 增加数据增强，减少训练轮数，调低 LoRA 权重 |
| 平台对接困难 | 前端无法调用模型 | 先做假接口，再接入真实推理 |
| 项目范围过大 | 进度拖延 | 先完成最小闭环，再扩展功能 |
