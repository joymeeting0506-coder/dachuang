# AI赋能的非遗纹样智能生成与商用设计平台

本项目面向大学生创新训练计划，目标是做出一个可以展示、可以实验、可以在校内场景验证的“非遗纹样智能生成与商用设计平台”原型。项目研究范围聚焦三条主线：

1. 整合与优化非遗纹样公开数据集，覆盖剪纸、敦煌壁画、苗绣、传统丝绸纹样等类别，完成数据清洗、标准化处理和风格标签标注。
2. 选型并改进生成模型，重点尝试 StyleGAN2、Stable Diffusion XL（SDXL）+ LoRA 微调，提升纹样风格还原度和生成多样性，探索小样本场景下的模型收敛方案。
3. 开发商用设计平台原型，实现纹样生成、风格定制、参数调节、格式导出等功能，适配文创设计需求，并在校内场景验证可用性。

## 一、最终要做成什么

项目最终建议交付以下内容：

- 一个整理好的非遗纹样数据集，包含原图、清洗后图片、标签文件、数据说明文档。
- 一套可复现实验流程，包含数据预处理脚本、模型训练或微调脚本、推理生成脚本。
- 两类模型结果：
  - StyleGAN2 路线：用于理解传统 GAN 生成纹样的流程和效果。
  - SDXL + LoRA 路线：作为主要展示路线，用较低成本微调出特定非遗风格。
- 一个 Web 原型平台，用户可以选择纹样类别、输入提示词、调节参数、生成图片并导出。
- 一份项目报告，包含技术路线、实验对比、平台截图、用户测试反馈和改进方向。

## 二、学生小白实现路线

建议不要一开始就追求“完整商用平台”，先把最小闭环跑通：数据集 -> LoRA 微调 -> 生成图片 -> Web 页面调用生成接口。之后再逐步加模型对比、参数调节和导出功能。

### 第 0 步：环境准备

每位成员先完成基础环境安装：

- Git：用于代码版本管理。
- Python 3.10 或 3.11：用于数据处理和模型训练。
- Conda 或 venv：用于管理 Python 环境。
- PyTorch：根据服务器 CUDA 版本安装。
- VS Code 或 PyCharm：用于代码开发。
- Node.js 20+：用于前端开发。

推荐先建立统一环境：

```bash
conda create -n heritage-pattern python=3.10
conda activate heritage-pattern
pip install torch torchvision torchaudio
pip install diffusers transformers accelerate peft safetensors datasets pillow opencv-python pandas tqdm fastapi uvicorn
```

前端推荐使用 React 或 Vue。初学者如果没有明显偏好，建议使用 React + Vite，因为启动快、资料多、原型开发方便。

```bash
npm create vite@latest frontend
cd frontend
npm install
npm run dev
```

### 第 1 步：收集公开数据

先不要追求数量特别大，建议每类先收集 100-300 张图片，跑通流程后再扩展。

优先收集这些类别：

- 剪纸纹样：人物、动物、花鸟、窗花、节庆图案。
- 敦煌壁画纹样：飞天、藻井、莲花、边饰、云纹。
- 苗绣纹样：几何纹、蝴蝶纹、鸟纹、花草纹。
- 传统丝绸纹样：团花、缠枝、云纹、龙凤、植物纹。

数据来源可以包括公开数据集、博物馆开放资源、论文附带数据、公开图片资料等。注意记录来源，避免使用版权不清晰的商用素材。

每张图片建议至少记录以下信息：

```csv
image_id,file_path,category,style,theme,color,source,license,quality_score
000001,data/raw/papercut/000001.jpg,papercut,folk,flower,red,xxx,public,4
```

字段说明：

- `category`：大类，如 `papercut`、`dunhuang`、`miaoxiu`、`silk`。
- `style`：风格，如 `folk`、`religious`、`geometric`、`court`。
- `theme`：主题，如 `flower`、`bird`、`cloud`、`dragon`。
- `color`：主色，如 `red`、`gold`、`blue_green`。
- `source`：图片来源。
- `license`：版权或开放协议。
- `quality_score`：人工质量评分，1-5 分。

### 第 2 步：数据清洗与标准化

这一阶段目标是让模型看到“干净、统一、可训练”的图片。

需要处理的问题：

- 删除重复图、模糊图、分辨率过低图。
- 去除明显水印、边框、文字说明。
- 裁剪主体区域，让纹样尽量占据画面中心。
- 统一尺寸，例如 512x512 或 1024x1024。
- 统一图片格式，例如全部转成 PNG 或 JPG。
- 为每张图补充风格标签和描述文本。

建议的数据处理流程：

```text
原始图片
  -> 去重
  -> 人工筛选
  -> 裁剪主体
  -> 调整尺寸
  -> 生成标签 CSV/JSON
  -> 形成训练集、验证集、测试集
```

初学者可以先手动筛选一小批高质量数据，再写脚本批量处理。不要一上来就把所有数据自动化，否则很容易把脏数据带进模型。

### 第 3 步：先跑通 SDXL + LoRA

SDXL + LoRA 是本项目最推荐的主路线。原因是它对小样本更友好，训练成本比从零训练扩散模型低，生成效果也更适合展示。

最小实验方案：

1. 每个风格先准备 50-100 张高质量图片。
2. 每张图片配一条简短描述，例如：

```text
a traditional Chinese papercut pattern, red paper, symmetrical flower motif
a Dunhuang mural decorative pattern, lotus and cloud motif, ancient Chinese style
a Miao embroidery pattern, geometric butterfly motif, colorful textile texture
```

3. 使用 LoRA 微调 SDXL。
4. 用固定提示词生成图片，观察是否学到了风格。
5. 调整训练轮数、学习率、LoRA rank、数据质量。

建议先训练单一风格，例如剪纸。剪纸风格颜色鲜明、结构清楚，最容易观察模型有没有学到特征。单风格跑通后，再做敦煌、苗绣、丝绸等多风格。

可记录的实验指标：

- 风格还原度：人工评分 1-5 分。
- 图案清晰度：是否有明显糊图、断裂、畸形。
- 多样性：同一提示词多次生成是否过于相似。
- 可用性：是否适合印在明信片、帆布包、书签、手机壳等文创产品上。

### 第 4 步：补充 StyleGAN2 对比实验

StyleGAN2 可以作为对比路线，不建议作为平台主力模型。它适合理解 GAN 如何学习纹样分布，也适合展示“传统生成模型”和“扩散模型微调”的差异。

建议做法：

1. 选择一个视觉差异明显的类别，例如剪纸或丝绸纹样。
2. 将图片统一到 256x256 或 512x512。
3. 使用开源 StyleGAN2-ADA 代码训练。
4. 保存生成样例和训练曲线。
5. 与 SDXL + LoRA 结果对比。

对比维度：

- 小样本训练是否稳定。
- 生成图片是否清晰。
- 是否能按提示词控制主题和颜色。
- 是否适合作为交互式平台后端。

预期结论通常是：StyleGAN2 可以生成某类纹样，但可控性较弱；SDXL + LoRA 更适合“输入需求 -> 生成设计图”的平台应用。

### 第 5 步：开发后端生成服务

后端负责接收前端参数，调用模型生成图片，并返回图片地址。

建议使用 FastAPI，初学者更容易跑通。后续如果课程或团队要求 Java，也可以再用 SpringBoot 封装业务接口。

核心接口建议：

```text
POST /api/generate
  输入：类别、风格、提示词、颜色、尺寸、随机种子、生成数量
  输出：生成图片 URL、参数记录、生成时间

GET /api/images
  输出：历史生成结果列表

GET /api/styles
  输出：可选风格和 LoRA 模型列表

POST /api/export
  输入：图片 ID、导出格式
  输出：PNG/JPG/SVG/PDF 文件地址
```

后端第一版可以不接真实大模型，先用本地示例图片模拟返回，保证前端流程完整。等模型推理脚本稳定后，再替换成真实生成接口。

### 第 6 步：开发前端设计平台

平台第一屏应直接是可用工具，而不是介绍页。建议包含以下区域：

- 左侧参数面板：类别、风格、主题、颜色、尺寸、生成数量、随机种子。
- 中间生成区：展示生成结果和加载状态。
- 右侧历史记录或详情面板：展示当前图片参数、评分、导出按钮。
- 顶部工具栏：项目名称、保存、导出、重置等操作。

最小功能闭环：

1. 用户选择类别：剪纸、敦煌、苗绣、丝绸。
2. 用户输入提示词或选择主题。
3. 用户调节参数，例如颜色、尺寸、生成数量。
4. 点击生成。
5. 页面展示生成结果。
6. 用户选择图片并导出 PNG/JPG。

进阶功能：

- 风格强度滑块：控制 LoRA 权重。
- 多图对比：一次生成 4 张，方便挑选。
- 收藏夹：保存优秀结果。
- 二次编辑：基于已有图片继续生成变体。
- 文创预览：将纹样贴到书签、帆布包、明信片、手机壳模板上。

### 第 7 步：校内场景可用性验证

项目不需要真的商业上线，但要证明“有用、能用、适合文创设计”。

可以选择 10-20 名校内同学进行测试：

- 让用户生成一张适合校园文创的非遗纹样。
- 观察用户能否独立完成生成和导出。
- 记录用户选择最多的风格和参数。
- 收集用户对美观度、操作难度、实用性的评分。

测试问卷可以包含：

- 生成结果是否具有非遗风格？
- 操作流程是否容易理解？
- 参数调节是否有帮助？
- 是否愿意将生成图用于文创设计？
- 希望增加哪些功能？

## 六、关键技术实现说明

### 数据标签如何设计

标签不要太复杂，初版只需要满足模型训练和平台筛选。

推荐标签：

```json
{
  "image_id": "papercut_0001",
  "category": "papercut",
  "style": "folk",
  "theme": ["flower", "symmetry"],
  "main_color": "red",
  "description": "traditional Chinese papercut pattern, red paper, symmetrical flower motif",
  "source": "public dataset or museum open resource",
  "quality_score": 4
}
```

### LoRA 微调如何理解

可以把 LoRA 理解成“不给大模型重新上大学，只给它补一门非遗纹样选修课”。我们不需要从零训练 SDXL，只需要用少量高质量纹样图，让模型学会某种风格。

影响效果的主要因素：

- 数据质量比数据数量更重要。
- 图片描述要稳定，不要一张图一种完全不同的写法。
- 训练轮数过少会学不到风格，过多会过拟合。
- LoRA 权重过高会风格强但容易僵硬，过低会风格不明显。

### 小样本问题怎么处理

本项目很可能遇到每类数据不够多的问题，可以这样解决：

- 优先人工筛选高质量图片。
- 做轻量数据增强，例如水平翻转、随机裁剪、颜色轻微扰动。
- 先训练单类风格，不要一开始混合太多类别。
- 使用 LoRA 而不是从零训练大模型。
- 记录失败结果，把失败案例作为实验分析的一部分。

### 平台如何对接模型

平台调用模型可以分三步走：

1. 假接口阶段：前端点击生成后，后端随机返回几张已有样例图。
2. 脚本调用阶段：后端调用本地 Python 推理脚本生成图片。
3. 服务化阶段：模型常驻 GPU，后端通过接口发送参数，减少每次加载模型的时间。

第一版能演示即可，不要过早追求高并发和复杂部署。

## 七、建议优先完成的最小版本

如果时间紧，优先做这个版本：

- 数据：只做剪纸和敦煌两个类别，每类 100 张左右。
- 模型：只做 SDXL + LoRA，StyleGAN2 放到后续对比。
- 平台：只实现类别选择、提示词输入、生成图片、PNG 导出。
- 测试：找 10 个同学试用并填写问卷。

这个版本已经能支撑大创中期检查和阶段展示。后续再逐步扩展苗绣、丝绸纹样、文创预览、多模型对比等功能。

## 八、常见风险与解决办法

| 风险 | 表现 | 解决办法 |
| --- | --- | --- |
| 数据版权不清晰 | 图片不能公开展示或写进成果 | 优先使用开放资源，保留来源记录 |
| 数据质量不高 | 生成结果模糊、风格混乱 | 人工筛选高质量图，减少杂图 |
| 显存不够 | SDXL 训练报错或速度很慢 | 降低分辨率，减小 batch size，使用 LoRA 和混合精度 |
| 小样本过拟合 | 生成图高度相似 | 增加数据增强，减少训练轮数，调低 LoRA 权重 |
| 前后端对接困难 | 页面有了但不能生成 | 先做假接口，再逐步接入真实推理 |
| 项目范围过大 | 每条线都做不深 | 先完成最小闭环，再扩展功能 |

## 九、验收标准

项目完成时至少应满足：

- 数据集有清晰目录、标签和来源说明。
- 至少完成一个非遗类别的 LoRA 微调并能稳定生成纹样。
- 至少完成一组 StyleGAN2 或其他模型的对比实验。
- Web 平台可以完成生成、参数调节和图片导出。
- 有用户测试记录和结果分析。
- 项目报告能说明技术路线、实验过程、平台实现和不足之处。

## 十、推荐推进顺序

最推荐的实际执行顺序：

```text
先做数据样例

  -> 做前端假接口
  -> 后端接入真实推理
  -> 扩展多风格
  -> 补 StyleGAN2 对比
```
